## A* 泊车实验 Phase 5

Phase 5 第一版完成 learning heuristic 最小闭环：

```text
Hybrid A* 成功路径
-> state-goal 样本
-> cost_to_go 标签
-> MLP 学习 heuristic
-> tie-breaker 接回 Hybrid A*
```

主要结果：

- 数据集：100 次随机起点，100/100 成功，1783 条样本
- 离线评估：MAE `0.6216`，RMSE `1.0099`，MLP ranking acc `0.9832`
- 接回搜索：`tie_precision=0` 时 expanded nodes 减少约 `6.3%`
- 批量推理后 planning time 从 `3.2426s` 降到 `2.6277s`，但仍慢于 baseline

## Phase 6：Neural A* 2D Toy

Phase 6 先不直接做泊车 SE(2)，而是复现 2D learning heuristic 流程：

```text
随机 2D 栅格地图
-> Dijkstra cost-to-go 标签
-> CNN 学 heuristic map
-> 后续接回 A*
```

当前完成 500 张 `32x32` 随机地图生成、Dijkstra 标签生成、A* baseline 对比和第一版 CNN heuristic 训练。CNN test MAE 为 `7.2525`，当前还未接回 A* 做 expanded nodes 对比。

## 遇到的问题

- MLP heuristic 有排序价值，但 PyTorch 小 batch 推理开销较大
- 第一版 MLP 没有地图输入，不能真正理解障碍物结构
- 2D Dijkstra 不考虑车辆姿态、倒车和非完整约束，因此不一定适合泊车 cost_to_go
- CNN 在远离 goal 的区域误差更大，说明全局绕路结构仍难学习
